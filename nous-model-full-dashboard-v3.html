Методология и ограничения

Чтобы красивый интерфейс не создавал ложной точности, здесь раскрыта логика всех производных показателей.

Quality Index

Каждая доступная метрика нормализуется относительно диапазона в этом файле. Затем среднее слегка притягивается к нейтральным 50 с помощью двух виртуальных наблюдений — так один узкий benchmark не становится автоматически абсолютным лидером.

(Σ normalized scores + 2·50) / (coverage + 2)

Coverage по-прежнему показан рядом: широкий набор независимых тестов надёжнее одной удачной метрики.

Value Index

Сначала рассчитывается смешанная цена: 70% input и 30% output. Quality делится на логарифмический штраф цены, после чего лучший результат нормализуется до 100.

Quality / log₂(2 + 0.7·input + 0.3·output)

Это практическая эвристика для навигации, а не экономический стандарт.

Aliases и proxy

Preview/thinking/snapshot-варианты сопоставлены с ближайшим ID каталога только там, где связь очевидна. Benchmark-only модели не получают выдуманную цену.

StepFun Arena-результат помечен как proxy, поскольку в исходном документе он связан с allied step-3.5-flash.
